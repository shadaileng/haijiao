## 2. 请求流程

### 2.1 标准 API 请求

```
页面组件 → request() → fetch(/api/{path}) → Worker → haijiao.com
                                                    │
                                             响应 JSON (可能加密)
                                                    │
                                            ← decodeMultiBase64()
                                                    │
                                            ← toCamelCase() (snake_case → camelCase)
                                                    │
                                            ← JSON.parse()
                                                    │
                                            ← 返回给组件
```

### 2.2 图片请求（特殊解密）

```
loadImg() → fetch(直连 haijiao.com) → 获取加密字符串
                                         │
                                 ← customDecode()
                                         │
                                 ← base64 data URL
```

### 2.3 视频请求

```
loadVideoSrc() → fetch(直连 /api/attachment) → POST → haijiao.com
                     携带 X-User-Id / X-User-Token
                                                    │
                                            ← 解码后返回视频信息
                                                    │
                                            (含 keyPath / remoteUrl)
```

## 3. 加密解密链路

### 3.1 API 响应加密

```
haijiao.com 后端 → JSON.stringify(data)
                  → UTF-8 编码
                  → btoa() × 3（三重 Base64）
                  → 包装为 ApiResult.data
```

### 3.2 前端解密

```
data (string) → atob() × 3 → UTF-8 decode → JSON.parse → 原始数据
```

对应代码：`src/api/request.ts` 中 `decodeMultiBase64()` + `utf8Decode()`

### 3.3 snake_case → camelCase 自动转换

API 响应中的 snake_case 字段（如 `topic_id`、`create_time`）在 `request()` 返回前自动转换为 camelCase（如 `topicId`、`createTime`），与 TypeScript 接口类型保持一致。

对应代码：`src/utils/transform.ts` 中 `toCamelCase()`

### 3.4 图片自定义编码

图片使用非标准 Base64 变体编码：

```
字符集: ABCD*EFGHIJKLMNOPQRSTUVWX#YZabcdefghijklmnopqrstuvwxyz1234567890
         (标准 Base64 的 +/ 被替换为 *#，去掉 = 填充)
解码逻辑: 4 字符 → 3 字节（customDecode 函数）
```

## 4. 认证方式

### 4.1 请求头认证

```
X-User-Id: {uid}
X-User-Token: {token}
```

### 4.2 登录获取

```
POST /login/signin
  Body: { Username, Password, Sign: MD5(Username + Password + UA) }
  →
  Response: { token, user: { id, nickname, ... } }
  →
  存储: settings store (uid, token, nickname, apiBase) → localStorage
```

> 详见 [01-登录功能实施方案.md](../plans/01-登录功能实施方案.md)

## 6. 目录结构

```
src/
├── api/request.ts        # HTTP 请求封装、加解密、toCamelCase、所有 API
├── types/index.ts        # TypeScript 类型定义
├── stores/               # Pinia 状态 (settings/user/homepage/hot)
├── composables/          # useMirrorConfig、useClipboard
├── plugins/              # v-headicon、v-content 指令
├── components/           # Topics/TopicContent/Comment/UserInfo/TabBar
├── utils/                # image.ts(loadImg/customDecode)、transform.ts(toCamelCase)
├── views/                # 10 个页面组件
├── router/index.ts       # 10 条路由 + scrollBehavior
├── styles/               # 全局 SCSS
├── App.vue               # 根组件
└── main.ts               # 入口，注册插件
```
