## 1. 问题描述

无限滚动（`<van-list>`）在帖子列表中存在两个问题：

1. **滚动越多内存占用越大**：所有历史页面数据持续保留在内存中，不释放
2. **数据覆盖 bug**：`UserView` 和 `UserHomeView` 的 `pageto()` 函数用 `splice(0, length, ...data.results)` 替换而非追加数据（`SearchView` 使用 `push()` 追加，但同样存在内存问题）

根本解决方向是将所有帖子列表视图改用 `<van-pagination>` 分页按钮，每次只保留当前页数据。

## 2. 设计目标

1. 用 `<van-pagination>` 分页按钮替代 `<van-list>` 无限滚动
2. 翻页后自动滚动到列表顶部
3. 用 `sessionStorage` 记住当前页码（刷新页面不丢失）
4. 覆盖所有帖子列表视图：HotTopicsView（已完成）、UserView、UserHomeView、SearchView

## 3. 改动方案

### 3.1 `src/components/Topics.vue`

新增 `mode` prop 支持两种渲染模式：

**新增 Props：**

| Prop | 类型 | 默认值 | 说明 |
|:-----|:----:|:------:|------|
| `mode` | `'scroll' \| 'pagination'` | `'scroll'` | 渲染模式 |
| `pageIndex` | `number` | `1` | 当前页码（pagination 模式） |
| `totalItems` | `number` | `0` | 总条目数（pagination 模式） |
| `pageSize` | `number` | `20` | 每页条数（pagination 模式） |

**新增 Emits：**

| Event | Payload | 说明 |
|:------|:--------|------|
| `pageChange` | `page: number` | 页码切换（pagination 模式） |

**模板变更：**

```html
<!-- scroll 模式：保持现有 van-list，不改变其他视图 -->
<template v-if="mode === 'scroll'">
  <van-list ...>
    ...（现有卡片 v-for）...
  </van-list>
</template>

<!-- pagination 模式：直接渲染列表 + 分页按钮 -->
<template v-else>
  ...（卡片 v-for，van-cell 复用）...
  <div v-if="totalItems > pageSize" class="pagination-wrapper">
    <van-pagination
      :model-value="pageIndex"
      :total-items="totalItems"
      :items-per-page="pageSize"
      @change="(p: number) => emit('pageChange', p)"
    />
  </div>
</template>
```

卡片内容的 `<van-cell v-for="item in topics">` 模板是两种模式共用的，提取到 `<template>` 片段中避免重复。

**暴露的方法：**
- scroll 模式保持 `startLoad` / `endLoad` / `finishLoad`
- pagination 模式暴露空方法，确保 `defineExpose` 接口一致

### 3.2 `src/views/HotTopicsView.vue`

**状态替换：**

```typescript
// 删除
const topicsDom = ref()
const skeletonLoading = ref(true)

// 新增
const pageIndex = ref(Number(sessionStorage.getItem('hotPageIndex')) || 1)
const totalItems = ref(0)
const pageSize = 20
const loading = ref(true)
```

**函数替换：**

```typescript
const loadPage = async (page: number) => {
  loading.value = true
  const result = await api.hot({ params: { page, limit: pageSize } })
  if (!result.success) {
    showToast(result.message || '加载热门列表失败')
    loading.value = false
    return
  }
  topics.length = 0
  topics.push(...result.data.results)
  totalItems.value = result.data.page.total
  pageIndex.value = page
  sessionStorage.setItem('hotPageIndex', String(page))
  loading.value = false
  window.scrollTo({ top: 0, behavior: 'smooth' })
}
```

**onMounted 更新：**

```typescript
onMounted(() => {
  loadPage(pageIndex.value)
})
```

**模板更新：**

```html
<Topics
  mode="pagination"
  :topics="topics"
  :skeletonLoading="loading"
  :pageIndex="pageIndex"
  :totalItems="totalItems"
  :pageSize="pageSize"
  @pageChange="loadPage"
/>
```

### 3.3 `src/types/index.ts`

在 `Page` 接口中补充 `totalPage` 字段：

```typescript
export interface Page {
  index: number
  size: number
  total: number
  totalPage?: number
}
```

该字段由 `toCamelCase()` 从服务端 `total_page` 自动映射而来。

### 3.4 `src/views/UserView.vue`

**删除：**
- `topicsDom` ref（不再需要 van-list 暴露的方法）
- `skeletonLoading` ref
- `liteTopics` 响应式对象（`LiteTopicPage`）
- `pageto()` 函数
- `loadMore()` 函数

**新增：** `loadPage()` 在函数首尾管理 `loading`，确保每次翻页时 Topics 组件显示骨架屏。

```typescript
const pageIndex = ref(1)
const totalItems = ref(0)
const pageSize = 15
const loading = ref(true)

onMounted(async () => {
  await loadPage(1)
})

const loadPage = async (page: number) => {
  loading.value = true
  if (!userId.value) { loading.value = false; return }
  const resp = await api.topics({ params: { userId: userId.value, page, type: 1 } })
  if (!resp.success) {
    showToast(resp.message || '获取主题失败')
    loading.value = false
    return
  }
  const data = resp.data
  if (data?.results) {
    topics.length = 0
    topics.push(...data.results)
  }
  if (data?.page) {
    totalItems.value = data.page.total
    pageIndex.value = page
  }
  loading.value = false
}
```

**模板：** 直接传递 `skeletonLoading`。

```html
<Topics
  mode="pagination"
  :topics="topics"
  :skeletonLoading="loading"
  :pageIndex="pageIndex"
  :totalItems="totalItems"
  :pageSize="pageSize"
  @pageChange="loadPage"
/>
```

### 3.5 `src/views/UserHomeView.vue`

与 UserView 同理，额外注意 `watch` 中切换用户时重置 `pageIndex = 1`、`totalItems = 0`。

**删除：**
- `topicsDom` ref
- `skeletonLoading` ref
- `liteTopics` 响应式对象
- `pageto()` 函数
- `loadMore()` 函数
- 外层 `<template v-if="loading">` 自定义骨架屏块（改用 Topics 组件的 `van-skeleton`）

**新增：** `loadPage()` 管理 `loading`，UserInfo 在 `v-if="userInfo"` 下独立显示，翻页时不受影响。

```typescript
const pageIndex = ref(1)
const totalItems = ref(0)
const pageSize = 15
const loading = ref(true)

onMounted(async () => {
  if (userId.value) await loadUserInfo(userId.value)
  await loadPage(1)
})

watch(() => route.params.userId, async (newId) => {
  if (newId && newId !== userId.value) {
    userId.value = newId as string
    nickname.value = (route.params.nickname as string) || ''
    userInfo.value = null
    pageIndex.value = 1
    totalItems.value = 0
    await loadUserInfo(userId.value)
    await loadPage(1)
  }
})

const loadPage = async (page: number) => {
  loading.value = true
  if (!userId.value) { loading.value = false; return }
  const resp = await api.topics({ params: { userId: userId.value, page, type: 1 } })
  if (!resp.success) {
    showToast(resp.message || '获取主题失败')
    loading.value = false
    return
  }
  const data = resp.data
  if (data?.results) {
    topics.length = 0
    topics.push(...data.results)
  }
  if (data?.page) {
    totalItems.value = data.page.total
    pageIndex.value = page
  }
  loading.value = false
}
```

**模板更新：** 去掉外层 `v-if="loading"` 骨架屏，改用 Topics 的 `skeletonLoading` prop。

```html
<van-nav-bar ... />
<UserInfo v-if="userInfo" :userInfo="userInfo" />
<van-divider v-if="userInfo" />
<Topics
  mode="pagination"
  :topics="topics"
  :skeletonLoading="loading"
  :pageIndex="pageIndex"
  :totalItems="totalItems"
  :pageSize="pageSize"
  @pageChange="loadPage"
/>
```

### 3.6 `src/views/SearchView.vue`

搜索视图较为特殊：首次加载显示热门标签，搜索后才展示帖子列表。

**删除：**
- `topicsDom` ref（不再需要 van-list 暴露的加载状态方法）
- `index` ref（自增页码，改为 `pageIndex`）
- `search()` 中的 `push()` 追加逻辑（改为替换）
- `<template v-else-if="loading">` 自定义骨架屏块（改用 Topics 组件的 `van-skeleton`）
- `search()` 中手动管理 `loading` 的代码（由 `loadPage` 统一管理）

**新增：** `loadPage()` 管理 `loading`，首次搜索和翻页时骨架屏由 Topics 统一展示。

```typescript
const pageIndex = ref(1)
const totalItems = ref(0)
const pageSize = 20
const loading = ref(false)

const search = async (tag?: string) => {
  if (tag) key.value = tag
  if (!key.value) { showToast('请输入搜索关键词'); return }
  hasSearched.value = true
  await loadPage(1)
}

const loadPage = async (page: number) => {
  loading.value = true
  const result = await api.search({ params: { page, node_id: 0, key: key.value } })
  if (!result.success) {
    showToast(result.message || '搜索失败')
    loading.value = false
    return
  }
  if (result.data?.results) {
    topics.length = 0
    topics.push(...result.data.results)
  }
  if (result.data?.page) {
    totalItems.value = result.data.page.total
    pageIndex.value = page
  }
  loading.value = false
}

const onClear = () => {
  key.value = ''
  hasSearched.value = false
  topics.splice(0, topics.length)
  pageIndex.value = 1
  totalItems.value = 0
}
```

**模板更新：** 去掉中间的骨架屏块，统一由 Topics 的 `van-skeleton` 处理。

```html
<template v-if="!hasSearched">
  ...（搜索框 + 标签）...
</template>
<Topics
  v-else
  mode="pagination"
  :topics="topics"
  :skeletonLoading="loading"
  :pageIndex="pageIndex"
  :totalItems="totalItems"
  :pageSize="pageSize"
  @pageChange="loadPage"
/>
```

**key 变化时重置搜索：**

```typescript
watch(key, (val) => {
  if (!val && hasSearched.value) onClear()
})
```

## 4. 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| API 返回空结果 | `van-empty` 由 Topics 组件现有逻辑处理 |
| 总条目 < pageSize（只有一页） | `v-if="totalItems > pageSize"` 隐藏分页按钮 |
| 加载中 | `skeletonLoading` 传递 `true`，Topics 显示骨架屏 |
| 快速翻页 | `loading` 标志防止重复请求 |
| 请求失败 | Toast 提示 + `loading = false` |
| 刷新页面 | `sessionStorage` 恢复上次页码，直接请求对应页 |
| 跨导航返回 | `keep-alive` 缓存保持组件状态，`scrollBehavior` 恢复滚动位置 |
