## 1. 问题描述

用户主页（`/homepage/:userId/:nickname?`）打开过一次后，再去打开另一个用户的主页，会**先看到上一个用户的主页内容**（帖子列表 + 用户信息），然后才显示当前用户的主页内容。

### 复现步骤

1. 打开用户 A 主页，等待内容加载完成
2. 通过任意入口（关注列表、帖子中的用户链接等）跳转到用户 B 主页
3. 观察到用户 B 的主页短暂闪现用户 A 的内容后，再显示正确的用户 B 内容

## 2. 根因分析

### 2.1 keep-alive 复用组件实例

`App.vue:49` 将 `UserHomeView` 加入 `<keep-alive :include="[...]">`。当路由从 `/homepage/A` 变为 `/homepage/B` 时：

- **组件实例被保持**，不会销毁重建
- `onMounted` 不会再次执行
- 仅靠 `watch(() => route.params.userId, ...)` 响应参数变化

### 2.2 异步 watch 导致时序间隙

当前 `watch` 处理器（`src/views/UserHomeView.vue:32`）：

```
watch fires
  ├─ 同步阶段: userId='B', userInfo=null, pageIndex=1, totalItems=0
  │             但 topics[] 未清，loading 仍为 false
  ├─ Vue 触发一次重渲染 ← 此时 topics 还是 A 的数据，loading=false
  ├─ await loadUserInfo('B')     ← 网络请求，异步等待
  ├─ await loadPage(1)
  │    ├─ loading=true           ← 骨架屏出现
  │    ├─ topics.length=0        ← 清空旧帖子
  │    └─ 推入新数据 → 正确渲染
  └─ 完成
```

**关键问题**：同步阶段未清除 `topics` 和设置 `loading=true`，导致一次渲染脏数据。

## 3. 修复方案

### 3.1 修改内容

**文件**：`src/views/UserHomeView.vue`

在 watch 处理器的同步阶段增加两行：

```ts
topics.length = 0    // 立即清空旧帖子列表
loading.value = true // 立即显示骨架屏
```

### 3.2 修改后时序

```
watch fires
  ├─ 同步阶段: userId='B', topics.length=0, loading=true,
  │             userInfo=null, pageIndex=1, totalItems=0
  ├─ Vue 渲染骨架屏 + 空列表  ← 无旧内容闪现
  ├─ await loadUserInfo + loadPage → 推入新数据
  └─ 正确显示用户 B 主页
```

### 3.3 改动量

- 1 个文件，新增 2 行代码
- 不改动其他组件、路由、store

## 4. 影响范围分析

| 场景 | 影响 | 说明 |
|:----|:----|:-----|
| `/homepage/A` → `/homepage/B` | ✅ 修复 | watch 同步清数据后渲染骨架屏 |
| `/homepage/A` → `/hot` → 返回 `/homepage/A` | ❌ 不影响 | 组件从缓存恢复，`route.params.userId` 不变，watch 不触发 |
| `/homepage/A` → `/hot` → 返回 `/homepage/B` | ✅ 修复 | watch 触发，同场景 1 |
| 首次进入任意用户主页 | ❌ 不影响 | `onMounted` 执行，watch 不触发 |

`loadPage` 内部后续也会执行 `topics.length = 0` 和 `loading.value = true`，覆盖赋值无副作用。

## 5. 验证方法

1. 本地启动：`pnpm run dev`
2. 打开用户 A 主页，等待加载完成
3. 点击帖子或关注列表中的用户 B 链接
4. 观察过渡过程中是否闪现用户 A 的内容
5. 类型检查：`pnpm run build`
