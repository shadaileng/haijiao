## 1. 背景

帖子详情（TopicView）的评论区目前仅展示评论本身，每条评论下的回复列表渲染存在以下问题：

- **回复内容未使用 `v-content` 指令**：`commend_item.content` 以纯文本展示，其中的 HTML 标签和附件（图片、视频等）无法被渲染
- **用户导航链接错误**：点击回复昵称跳转至父评论用户的个人主页（`item.userId`），而非回复作者的主页（`commend_item.userId`）
- **缺少头像展示**：回复列表仅显示昵称文本，缺少作者头像
- **不支持嵌套回复**：`CommentItem.commendList` 类型为 `CommentItem[]`（可递归嵌套），当前只渲染一层

## 2. 目标

1. 使用 `v-content` 指令渲染回复内容，使图片/视频/音频等附件正常展示
2. 修复用户导航：点击回复昵称跳转到回复作者的主页
3. 为每条回复添加头像（`v-headicon` 懒加载解码）
4. 支持递归嵌套回复（回复的回复）

## 3. 设计方案

### 3.1 新增 `src/components/ReplyList.vue`

递归组件，核心模板结构：

```
ReplyList (props: replies, handleClick)
  ├── .reply_item
  │   ├── van-image (头像, v-headicon)
  │   ├── .reply_body
  │   │   ├── nickname (点击跳转 reply.userId)
  │   │   ├── v-content="{content, attachments, handleClick}"
  │   │   └── ReplyList (递归, 若 reply.commendList?.length)
  │   └── ...
  └── ...
```

**关键设计决策：**

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 递归方式 | 组件自引用（`defineOptions name`） | 原生 Vue 3 SFC 支持，无需额外注册 |
| handleClick 传递 | prop 逐层传递 | 职责清晰，不依赖注入上下文 |
| 嵌套缩进 | CSS `padding-left: 1em * level` | 层级清晰，无需额外 state |
| 图片懒加载 | `v-content` 自带的 IntersectionObserver | 复用现有懒加载机制 |

### 3.2 修改 `src/components/Comment.vue`

替换第 42-51 行内联回复代码为 `<ReplyList>` 组件调用。

**改动范围：**
- `<script>` 区域：添加 `import ReplyList from './ReplyList.vue'`
- `<template>` 区域：`v-if="item.commendList?.length"` 块改为 `<ReplyList>`

**兼容性：** 不改变 API 调用逻辑、props 结构、store 状态，仅替换渲染方式。

## 5. 风险与注意事项

- **API 返回空数据**：若后端 `reply_list` 接口未返回 `commendList` 或返回空数组，组件不渲染，不影响已有功能
- **深度嵌套性能**：回复嵌套通常不超过 3 层，递归渲染性能可接受；极端嵌套时 CSS 缩进持续累积可能导致布局溢出，属正常边界
- **attachments 字段**：回复的 `attachments` 是 `string` 类型，`v-content` 中 `buildAttaMap` 已处理非数组情况（返回空 Map），嵌入 HTML 中的 `<img>` 仍能正常渲染
- **video 支持**：回复中的视频播放依赖 `topicId`；当前 `v-content` 对评论内容的视频处理已通过 `<video>` → 可点击 div 方式实现，回复中同理
