## 1. 现状

当前项目 10 个页面 + 6 个组件中，绝大部分已是数据驱动模式（API → reactive ref → 模板渲染），但仍存在以下问题：

| 类别 | 问题 | 位置 |
|:----|------|:-----|
| **硬编码** | Tab 栏 4 项固定不可配 | `TabBar.vue` |
| **硬编码** | 头像占位图用死 URL | `FollowView.vue` |
| **死 Store** | 定义并持久化但未被任何视图引用 | `stores/hot.ts`、`stores/homepage.ts` |
| **缺骨架屏** | API 加载期间用户看到空白 | `UserView.vue`、`SettingsView.vue` |
| **逻辑空转** | `van-list@load` 绑定了空函数 | `Comment.vue` |
| **类型丢失** | inject 返回 `any`，prop 用 `Object` | `SearchView.vue`、`UserInfo.vue` |
| **注入不一致** | 仅有 SearchView 用 `inject('$api')`，其余直接 import | 全局 |

---

## 2. 目标

1. 消除所有硬编码 UI 数据
2. 清理死 Store 代码
3. 补齐缺失的骨架屏加载态
4. 修复逻辑空转和类型安全问题
5. 统一数据注入模式

---

## 3. 实施方案

### 阶段一：消除硬编码

| 步骤 | 文件 | 操作 |
|:----:|:-----|:-----|
| 1 | `src/components/common/TabBar.vue` | Tab 配置改为响应式数组或 store 驱动，支持动态增删 tab |
| 2 | `src/components/common/TabBar.vue` | 根据登录态显示/隐藏「关注」tab（未登录时隐藏或置灰） |
| 3 | `src/views/FollowView.vue` | 移除 `cat.jpeg` 硬编码占位图，`v-headicon` 足以兜底 |

### 阶段二：清理死代码

| 步骤 | 文件 | 操作 |
|:----:|:-----|:-----|
| 4 | `src/stores/hot.ts` | 删除整个文件，移除 `stores/index` 导出 |
| 5 | `src/stores/homepage.ts` | 同上 |
| 6 | — | 验证构建无报错 |

### 阶段三：补齐骨架屏

| 步骤 | 文件 | 操作 |
|:----:|:-----|:-----|
| 7 | `src/views/UserView.vue` | 新增 `skeletonLoading` ref，API 加载中显示骨架卡片 |
| 8 | `src/views/SettingsView.vue` | `fetchCurrent()` 期间显示骨架屏，避免闪旧数据 |

### 阶段四：修复逻辑与类型

| 步骤 | 文件 | 操作 |
|:----:|:-----|:-----|
| 9 | `src/components/Comment.vue` | 删除空 `onLoad()`，`van-list` 改为普通 `div` 容器，使用 `<van-pagination>` 控制分页 |
| 10 | `src/views/SearchView.vue` | 为 `inject('$api')` 定义接口类型，消除 `any` |
| 11 | `src/components/UserInfo.vue` | prop 类型 `Object as any` 改为 `User` 类型 |

### 阶段五：统一注入模式（可选，影响面大）

| 步骤 | 文件 | 操作 |
|:----:|:-----|:-----|
| 12 | `src/api/request.ts` | 提取 `Api` 接口类型定义，供 inject 使用 |
| 13 | 各页面视图 | 统一使用 `inject('$api')` 或统一使用 `import { api }`，二选一 |

> 步骤 12-13 影响面较大，建议阶段性实施或暂缓。当前直接 import 模式并无功能缺陷。

## 5. 验收标准

1. `pnpm run build` 通过，无类型错误
2. TabBar 各 tab 点击导航正常
3. 未登录时「关注」tab 隐藏或提示登录
4. UserView 加载中显示骨架卡片
5. SettingsView 加载用户信息时不闪旧数据
6. 评论区分页正常，无空函数调用
7. SearchView api 调用处有类型提示
8. 删除的两个 store 无残留引用
