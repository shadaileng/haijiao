## 1. 背景

### 当前代理架构

| 层级 | 代码位置 | 当前值 | 是否可配置 |
|:-----|:---------|:-------|:----------|
| 前端 API_BASE | `src/api/request.ts:4` | `'/api'`（硬编码） | ❌ |
| Vite 开发代理 | `vite.config.ts:15` | `'https://haijiao.com'`（硬编码） | ❌ |
| Worker 环境变量 | `wrangler.toml:14` | `'https://haijiao.com'`（部署时设置） | 部署时可配 |
| 前端直连数据源 | `store.apiBase` | `'haijiao.com'`（用户可改） | ✅ 仅用于图片/视频直连 |

### 问题

`API_BASE = '/api'` 硬编码在 `request.ts` 中，所有 API 请求只能通过同域 Worker（或 Vite 开发代理）转发。当用户使用自定义部署的 Worker 或第三方代理服务时，无法将请求指向其他地址。

### 触发场景

- 用户部署了自己的 Worker 到自定义域名
- 默认代理地址因网络原因不可用，需要切换到备用地址
- 开发阶段需要将 API 指向其他环境

### 关键约束：登录页也需要暴露配置入口

登录页（`LoginView`）是路由守卫中的公开页面（`publicPages`），未登录用户只能访问登录页和设置页。如果默认代理地址不可用，用户无法登录、也无法进入设置页修改，因此 **登录页必须提供代理地址配置入口**。

## 2. 设计方案

### 2.1 数据模型

在 `user` store 中新增 `proxyBase` 字段：

| 字段 | 类型 | 默认值 | 说明 |
|:-----|:----|:-------|:-----|
| `proxyBase` | `string` | `''` | 代理地址；空串表示使用 `/api` |

持久化存储在 `localStorage` 的 `haijiao_user` 对象中。

### 2.2 请求层逻辑变更

将硬编码的 `const API_BASE = '/api'` 改为动态获取函数 `getProxyBase()`：

```
getProxyBase() → store.proxyBase 有值 ? 使用该值 : 回退 '/api'
```

`buildProxiedUrl()` 和 `login()` 中的前缀统一改为调用 `getProxyBase()`。

### 2.3 UI 交互

使用 `<van-dialog>`（非 `showDialog`），因其支持自定义 slot 内嵌 `<van-field>` 输入框。

**登录页**：在「手动配置 UID / Token」按钮下方新增「配置代理地址」按钮 → 弹出对话框 → 输入保存。

**首页**：在 feature grid 区域增加「代理地址」配置项，或在导航栏增加入口按钮 → 弹出同一对话框。

### 2.4 共享逻辑

创建 `src/composables/useProxyConfig.ts`，封装代理地址的读写和对话框状态，供两个页面复用。

## 3. 改动清单

### 3.1 `src/stores/user.ts`

| 变更 | 说明 |
|:-----|:------|
| 新增 `proxyBase` 状态 | `ref('')`，默认空串 |
| 新增 `setProxyBase(url: string)` | 更新值 + 调用 `saveUser()` 持久化 |
| `loadFromStorage()` | 从 localStorage 恢复 `proxyBase` |
| `logout()` | 重置 `proxyBase` 为 `''` |
| return 列表 | 暴露 `proxyBase` 和 `setProxyBase` |

### 3.2 `src/api/request.ts`

| 变更 | 说明 |
|:-----|:------|
| 删除 `const API_BASE = '/api'` | 替换为 `getProxyBase()` 函数 |
| `buildProxiedUrl()` | `/\`${API_BASE}\` → \`${getProxyBase()}\`` |
| `login()` | 同上，`\`${API_BASE}/login/signin\`` → `\`${getProxyBase()}/login/signin\`` |
| `getProxyBase()` | `store.proxyBase \|\| '/api'` |

### 3.3 `src/composables/useProxyConfig.ts`（新增）

```ts
export function useProxyConfig() {
  const store = useUserStore()
  const proxyUrl = ref(store.proxyBase)
  const showDialog = ref(false)
  const proxyDisplay = computed(() => store.proxyBase || '/api（默认）')

  function openConfig() { ... reset proxyUrl, showDialog = true }
  function saveConfig() { ... store.setProxyBase(), showDialog = false, Toast }

  return { proxyUrl, showDialog, proxyDisplay, openConfig, saveConfig }
}
```

### 3.4 `src/views/LoginView.vue`

在「手动配置 UID / Token」按钮下方新增：

```html
<van-cell title="代理地址" :value="proxyDisplay" is-link @click="openConfig" />
```

页面模板中增加 `<van-dialog>`，绑定 composable 的对话框状态。

### 3.5 `src/views/HomeView.vue`

在 feature grid 中新增一项：

```html
<van-grid-item icon="link-o" text="代理地址" @click="openConfig" />
```

或作为独立 cell group 显示。模板中增加相同的 `<van-dialog>`。

## 4. 数据流

```
用户输入代理地址
    → store.setProxyBase(url)
        → localStorage ('haijiao_user' 对象)
    → request.ts 下次请求自动读取
        → getProxyBase()
            → store.proxyBase 有值 ? 配置值 : '/api'
    → buildProxiedUrl() / login() 使用新地址发起请求
```

---

## 5. 不修改的文件

| 文件 | 原因 |
|:-----|:------|
| `worker.ts` | Worker 仍监听 `/api` 路径，默认场景不变 |
| `wrangler.toml` | 同上，默认 HAIJIAO_API_BASE 不变 |
| `vite.config.ts` | 开发代理仅在默认 `/api` 时生效；配置自定义地址时直接请求目标地址 |
| `router/index.ts` | 路由和守卫逻辑不变 |
| `src/types/index.ts` | 无需新增类型 |

## 7. 验证方法

| 验证项 | 方法 | 预期 |
|:-------|:-----|:------|
| 默认行为不变 | 不配置代理地址，请求 API | 仍走 `/api` 路径 |
| 配置后生效 | 输入 `https://custom-proxy.com`，查看网络请求 | 请求发往 `https://custom-proxy.com/...` |
| 清空后回退 | 清空代理地址 | 请求回退到 `/api` |
| 持久化 | 刷新页面后查看 | 代理地址值从 localStorage 恢复 |
| 登录页配置 | 未登录状态配置代理地址 | 登录请求发往新地址 |
