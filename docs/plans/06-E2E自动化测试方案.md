## 1. 背景与目标

### 1.1 现状
- 方案 05 已完成 `src/` 重建（视频播放、模块化状态、可配置镜像源 `X-Backend`、Worker 代理）。
- 项目 `devDependencies` 已含 `@playwright/test`，但**尚未编写任何测试**，也无本地 E2E 验证手段。
- 此前本地 `wrangler dev` 因 `workerd` 的 postinstall 在环境中受限无法安装，本地联调受阻。
- 关键约束：**官方域名 `haijiao.com` 在国内被屏蔽**；真正的可用地址是后台动态提供的、国内可访问的「镜像 / 数据源」域名，即配置页「镜像源（数据源地址）」字段（`settings.apiBase`）。代码默认 `https://haijiao.com` 仅作兜底占位，在 CN 实际不可达。

### 1.2 目标
1. 用 Playwright 驱动本机已安装的 Google Chrome（`C:\Program Files\Google\Chrome\Application\chrome.exe`），对应用做端到端自动化测试。
2. 覆盖四层：公开页冒烟、登录态链路、帖子详情（视频+图片）、镜像源 / 数据源 `X-Backend` 头断言。
3. 本地测试**使用 vite `server.proxy`**（仅 `npm run dev` 生效，不影响 CF 部署；生产仍由 `worker.ts` 代理），规避 `workerd` 安装受限问题。代理目标由配置页「数据源字段」经 `X-Backend` 头动态决定，与 `worker.ts` 行为对齐。

## 2. 环境分工（本地 vs 生产）

| 场景 | 代理方式 | 说明 |
|------|---------|------|
| **本地 E2E / 开发** | vite `server.proxy` | `npm run dev` 时 `/api` 由 vite 服务端代理；`router(req)` 读取请求头 `X-Backend`（应用按配置页「数据源字段」发出）→ 转发到对应镜像源；请求走同源 `/api`，规避 CORS |
| **生产部署** | Cloudflare Worker | `worker.ts` 解析 `X-Backend` 头选择镜像源，回退 `HAIJIAO_API_BASE` |

> 关键：`vite.config.ts` 的 `server.proxy` **只在 `npm run dev` 时生效**，`npm run build` 产物由 Worker 托管，不受该配置影响。这与方案 05「不使用 vite proxy」的原则不冲突——05 针对生产架构，本方案针对本地测试，二者分工明确。
>
> 重要：`haijiao.com` 在 CN 被屏蔽，本地代理默认回退到它同样不可达。因此**所有本地规格运行前，都必须先把配置页「数据源字段」设为后台提供的可用镜像域名**（经 `X-Backend` 头传递给 vite proxy），公开页冒烟也不例外。

## 3. 工具与浏览器

- **测试框架**：`@playwright/test`（已为 devDependency，无需再装）。
- **浏览器**：使用本机 Google Chrome，**无需下载 Playwright 自带 Chromium**。两种方式均可：
  - 方式 A（指定路径）：`launchOptions.executablePath = 'C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe'`
  - 方式 B（自动识别）：`launchOptions.channel = 'chrome'`
- **无头模式**：`headless: true`（CI/脚本友好）；视频测试只需断言 DPlayer 装配与 `<video>` 的 `src` 被设为 `.m3u8`，不要求真实解码出画面。

## 4. 配置改动

### 4.1 `vite.config.ts`（新增 `server.proxy`，router 读 `X-Backend`）
```ts
server: {
  port: 3000,
  host: '0.0.0.0',
  proxy: {
    '/api': {
      target: 'https://haijiao.com', // 仅作 changeOrigin 占位，实际目标由 router 决定
      changeOrigin: true,
      secure: false,
      router: (req) => {
        const backend = (req.headers['x-backend'] as string) || ''
        try {
          const u = new URL(backend)
          if (u.protocol === 'https:') return u.origin
        } catch { /* ignore */ }
        return 'https://haijiao.com'
      },
      rewrite: (p) => p.replace(/^\/api/, ''),
    },
  },
},
```
行为对齐 `worker.ts` 的 `resolveBackend`：`/api/topic/123` → `${X-Backend}/topic/123`；`X-Backend` 缺失/非法时回退 `https://haijiao.com`（CN 不可达，仅占位）。应用侧 `request.ts` 已对所有 `/api/*` 请求头写入 `X-Backend: settings.apiBase`，因此 vite proxy 会按配置页「数据源字段」转发。**不使用任何 env 变量**。

### 4.2 `playwright.config.ts`（新建）
```ts
import { defineConfig } from '@playwright/test'

export default defineConfig({
  testDir: './e2e',
  timeout: 30000,
  use: {
    baseURL: 'http://localhost:3000',
    headless: true,
    launchOptions: {
      executablePath: 'C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe',
      // 或 channel: 'chrome'
    },
  },
  webServer: {
    command: 'npm run dev',
    port: 3000,
    reuseExistingServer: true,
  },
})
```

### 4.3 `package.json`（新增脚本）
```json
"test:e2e": "playwright test"
```

## 5. 测试规格设计（`e2e/` 目录）

> 所有规格前置：测试启动时通过 `addInitScript` 注入持久化 `settings`，将 `apiBase` 设为**后台提供的可用镜像域名**（见 §6）。`mirror.spec` 额外验证该字段→`X-Backend`→代理目标的链路本身。

### 5.1 `public.spec.ts` — 公开页冒烟（需镜像域名，无需登录凭据）
| 用例 | 断言 |
|------|------|
| 访问 `/hot` | 列表渲染出帖子项（`.card` 存在），标题非空 |
| TabBar 跳转 | 点击「搜索」→ URL 变为 `/search` |
| 搜索 | 输入关键词回车，结果列表非空 |
| 图片解码 | 帖子封面 `img` 的 `src` 被替换为 data URI（非占位图） |

### 5.2 `auth.spec.ts` — 登录态链路（真实登录流程）
- 通过 UI 填入 `username`/`password` → 点击登录 → 等待跳转离开 `/login`。
- 登录成功后 `settings` store 自动写入 `uid`/`token`，无需手动注入。
- 用例：
  | 用例 | 断言 |
  |------|------|
  | 登录并验证 localStorage | 跳转 `/hot`，`settings.uid`/`token` 非空 |
  | 设置页 | 显示「已登录」 |
  | 关注列表 | `.card` 渲染 |
  | 用户主页 | 用户信息卡片 + 帖子列表渲染 |

### 5.3 `topic.spec.ts` — 帖子详情（登录 + 视频 + 图片，需凭据 + pid）
- 先走真实登录流程（同 `auth.spec.ts`）。
- 进入指定 `pid` 帖子详情。
- 断言：
  - 点击视频封面 `.hv-video-div` → `<video>` 的 `src` 以 `.m3u8` 结尾；
  - 帖子内容中 `<img>` 的 `src` 被替换为 data URI（图片解码成功）。

### 5.4 `mirror.spec.ts` — 镜像源 / 数据源 `X-Backend` 头
- `addInitScript` 注入自定义 `apiBase`（如 `https://mirror.example.com`，须为合法 `https:`）。
- 监听请求：`page.on('request', req => ...)`。
- 断言：应用发出的 `/api/*` 请求携带请求头 `X-Backend: https://mirror.example.com`，且 vite proxy 据此转发（可通过镜像端返回特征响应验证，或仅断言请求头）。

## 6. 凭据与参数管理

视频层与登录态测试需要**真实参数**（用户尚未提供），统一放在 `e2e/config.ts` 常量，由用户填充：

```ts
export const E2E = {
  mirrorDomain: '',   // 必填：后台提供的、国内可访问的镜像/数据源域名（含 https://）
  uid: '',            // 或提供 username / password
  token: '',
  username: '',
  password: '',
  videoPid: '',       // 一个 attachments 含 category==='video' 的真实帖子 ID
}
```

- `mirrorDomain` 为**必填**：本地代理目标完全由它经 `X-Backend` 决定；不填则所有规格（含公开页）在 CN 下不可达。
- 若提供 `username/password`，`auth.spec.ts` 可改用 `/api/login/signin` 真实登录流程；否则用 `addInitScript` 注入 `settings` 持久化状态绕过登录守卫。
- `videoPid` 或 `uid/token` 缺失时，对应 `auth/video` 规格标记为 `test.skip`，其余规格不受影响。

## 8. 风险与注意

- **无头 Chrome 下视频不保证真实解码出画面**：测试仅断言 DPlayer 装配与 `src` 设置，符合冒烟定位。
- **凭据依赖**：`auth/video` 层需要有效 `uid/token` 或账号密码，以及含视频的 `pid`；用户提供前对应规格先 skip。
- **不走 `wrangler dev`**：本方案本地走 vite proxy，规避此前 `workerd` postinstall 受限问题；生产部署仍依赖 Worker。
- **代理分工**：`vite.config.ts` 的 `server.proxy` 仅 `npm run dev` 生效，不进入 `dist/` 产物，生产架构（Worker 代理）保持不变。
- **X-Backend 校验**：Worker 仅接受合法 `https:` 域名作为镜像源；测试中注入的 `apiBase` / `mirrorDomain` 须为完整 `https://` 地址。
- **CN 屏蔽**：`haijiao.com` 国内不可达，所有本地规格均依赖配置页「数据源字段」填入可用镜像，不能依赖默认回退。
