## 1. 环境要求

| 工具 | 版本要求 |
|:----|:--------|
| Node.js | >= 18 |
| npm | >= 8 |
| Wrangler CLI | >= 3（部署用） |

## 2. 快速开始

```bash
# 安装依赖
npm install

# 开发服务器（支持热更新）
npm run dev

# 浏览器打开 http://localhost:3000
# API 代理自动转发到配置页「数据源」镜像域名
```

## 3. 可用命令

| 命令 | 说明 |
|:-----|------|
| `npm run dev` | 启动开发服务器（Vite dev server） |
| `npm run build` | 类型检查 + 构建生产包 |
| `npm run preview` | 预览构建结果 |
| `npm run test:e2e` | Playwright 端到端测试（驱动本机 Chrome） |
| `npm run cf:dev` | 本地模拟 Cloudflare Worker |
| `npm run cf:deploy` | 部署到 Cloudflare Workers |

### `npm run build` 执行流程

```
TypeScript 类型检查 (vue-tsc --noEmit)
      ↓
Vite 生产构建 (vite build)
      ↓
输出到 dist/
```

### `npm run cf:deploy` 执行流程

```
Vite 构建 (vite build)
      ↓
Wrangler 部署 (wrangler deploy)
      ↓
上传 dist/ + worker.ts → Cloudflare
```

## 4. 开发代理配置

本地开发通过 Vite 插件自定义中间件实现代理，`npm run dev` 时生效：

- 所有 `/api/*` 请求由自定义中间件拦截
- 读取请求头 `X-Backend`（应用按配置页「数据源字段」发出）动态转发到对应镜像源
- 未配置镜像源时回退 `HAIJIAO_API_BASE`（默认 `haijiao.com`，国内不可达）
- 生产部署由 `worker.ts` 代理，不受此配置影响

## 5. 构建配置

- 手动分块：`vue` 和 `vant` 拆分为独立 chunk
- 输出目录：`dist/`
- 部署前执行 `npm run build` 确保类型正确

## 6. 项目配置

| 文件 | 用途 |
|:-----|------|
| `vite.config.ts` | Vite 构建、自定义中间件代理、分块配置 |
| `tsconfig.json` | TypeScript 严格模式（strict: true） |
| `wrangler.toml` | Cloudflare Workers 部署配置 |
| `tailwind.config.js` | Tailwind CSS 定制 |
| `postcss.config.js` | PostCSS 配置 |
| `playwright.config.ts` | Playwright E2E 测试配置 |

## 7. 部署流程

### 7.1 首次部署

```bash
# 1. 安装 Wrangler
npm add -D wrangler

# 2. 登录 Cloudflare
npx wrangler login

# 3. 配置 wrangler.toml
#    - name: 应用名
#    - HAIJIAO_API_BASE: 环境变量

# 4. 部署
npm run cf:deploy
```

### 7.2 更新部署

```bash
npm run build && npm run cf:deploy
```

## 8. 代码规范

### 8.1 TypeScript

- 启用严格模式
- 所有组件使用 `<script setup lang="ts">`
- 类型定义放在 `src/types/index.ts`
- API 返回值必须定义接口类型

### 8.2 Vue 组件

- Composition API + `<script setup>`
- Props/Emits 使用 `defineProps` / `defineEmits`
- 组件命名 PascalCase

### 8.3 状态管理

- 使用 Pinia，模块化 store
- 持久化统一使用 `pinia-plugin-persistedstate` → localStorage
- Store 初始化时自动从 localStorage 恢复数据

### 8.4 API 请求

- 所有请求封装在 `api/request.ts`
- 通过 Cloudflare Worker 代理
- 支持 Base64 多层加密数据自动解密
- `request()` 对所有响应执行 `toCamelCase()` 转换
- 错误统一抛出 Error 对象

## 9. 常见问题

### 9.1 开发环境 API 返回 404

检查配置页「数据源」是否填入了可用的镜像域名。`haijiao.com` 国内被屏蔽，不可作为本地开发目标。

### 9.2 构建失败

```bash
# 检查 TypeScript 错误
npx vue-tsc --noEmit

# 清空缓存后重试
rm -rf dist node_modules/.vite
npm run build
```

### 9.3 部署后页面白屏

- 确认 `wrangler.toml` 中 `routes` 配置正确
- 确认 Worker 的 SPA 回落逻辑生效（非 `/api` 路径返回 `index.html`）

### 9.4 登录失败 — Sign 算法

登录使用 `js-md5` 库计算 Sign：`MD5(Username + Password + navigator.userAgent)`，输出 32 位小写 hex。

详见 [01-登录功能实施方案.md](../plans/01-登录功能实施方案.md) §3。
