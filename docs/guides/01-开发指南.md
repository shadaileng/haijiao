## 1. 环境要求

| 工具 | 版本要求 |
|:----|:--------|
| Node.js | >= 18 |
| pnpm | >= 8 |
| Wrangler CLI | >= 3（部署用） |

## 2. 快速开始

```bash
# 安装依赖
pnpm install

# 开发服务器（支持热更新）
pnpm dev

# 浏览器打开 http://localhost:5173
# API 代理自动转发到 https://haijiao.com
```

## 3. 可用命令

| 命令 | 说明 |
|:-----|------|
| `pnpm dev` | 启动开发服务器（Vite dev server） |
| `pnpm build` | 类型检查 + 构建生产包 |
| `pnpm preview` | 预览构建结果 |
| `pnpm cf:dev` | 本地模拟 Cloudflare Worker |
| `pnpm cf:deploy` | 部署到 Cloudflare Workers |

### `pnpm build` 执行流程

```
TypeScript 类型检查 (vue-tsc --noEmit)
      ↓
Vite 生产构建 (vite build)
      ↓
输出到 dist/
```

### `pnpm cf:deploy` 执行流程

```
Vite 构建 (vite build)
      ↓
Wrangler 部署 (wrangler deploy)
      ↓
上传 dist/ + worker.ts → Cloudflare
```

## 4. 开发代理配置

`vite.config.ts` 中配置了开发代理：

```typescript
server: {
  proxy: {
    '/api': {
      target: 'https://haijiao.com',
      changeOrigin: true,
    },
  },
}
```

开发环境所有 `/api/*` 请求直接转发到 `haijiao.com`，避免 CORS 问题。

## 5. 构建配置

- 手动分块：`vue` 和 `vant` 拆分为独立 chunk
- 输出目录：`dist/`
- 部署前执行 `pnpm build` 确保类型正确

## 6. 项目配置

| 文件 | 用途 |
|:-----|------|
| `vite.config.ts` | Vite 构建、代理、分块配置 |
| `tsconfig.json` | TypeScript 严格模式（strict: true） |
| `wrangler.toml` | Cloudflare Workers 部署配置 |
| `tailwind.config.js` | Tailwind CSS 定制 |
| `postcss.config.js` | PostCSS 配置 |
| `.env` / `.env.*` | 环境变量（.gitignore 中排除） |

## 7. 部署流程

### 7.1 首次部署

```bash
# 1. 安装 Wrangler
pnpm add -D wrangler

# 2. 登录 Cloudflare
npx wrangler login

# 3. 配置 wrangler.toml
#    - name: 应用名
#    - HAIJIAO_API_BASE: 环境变量

# 4. 部署
pnpm cf:deploy
```

### 7.2 更新部署

```bash
pnpm build && pnpm cf:deploy
```

## 8. 代码规范

### 8.1 TypeScript

- 启用严格模式
- 所有组件使用 `<script setup lang="ts">`
- 类型定义放在 `src/types/index.ts`
- API 返回值必须定义接口类型

### 8.2 Vue 组件

- Composition API + `<script setup>`
- Props/Emits 使用 `defineProps` / `defineEmits`
- 组件命名 PascalCase

### 8.3 状态管理

- 使用 Pinia，模块化 store
- 用户状态在 `stores/user.ts`
- 数据持久化使用 localStorage
- Store 初始化时自动从 localStorage 恢复数据

### 8.4 API 请求

- 所有请求封装在 `api/request.ts`
- 通过 Cloudflare Worker 代理
- 支持 Base64 多层加密数据自动解密
- 错误统一抛出 Error 对象

## 9. 常见问题

### 9.1 开发环境 API 返回 404

检查 `vite.config.ts` 代理配置是否正确，确认目标域名可达。

### 9.2 构建失败

```bash
# 检查 TypeScript 错误
npx vue-tsc --noEmit

# 清空缓存后重试
rm -rf dist node_modules/.vite
pnpm build
```

### 9.3 部署后页面白屏

- 确认 `wrangler.toml` 中 `routes` 配置正确
- 确认 Worker 的 SPA 回落逻辑生效（非 `/api` 路径返回 `index.html`）

### 9.4 登录失败 — Sign 算法

> **注意**：当前 `generateSign()` 为占位实现，直接将 password 原文传入。
> 需替换为正确的 `MD5(Username + Password + UserAgent)` 算法。

详见 [01-登录功能实施方案.md](../plans/01-登录功能实施方案.md) §3。
